Snake Game
##概要
本アプリケーションは、Javaを用いて実装したシンプルな「スネークゲーム」です。一定時間ごとにヘビが移動し、エサを取得することで身体が成長します。壁または自身の体に衝突した場合、ゲームオーバーとなる基本ロジックを実装しています。
---
##開発の動機
Java の学習過程において、前回数字猜测ゲームを無事に実装できたことを踏まえ、より深くクラスの責務分担やカプセル化の実際の使い方、GUI とイベント駆動、そしてプログラム全体がどのように動作するのかを理解するため、より代表的な例としてスネークゲームを選択しました。
実装の過程では、衝突判定、状態管理、イベントに基づく入力処理を初めて一通り実装しました。
本作品は、これまで学習してきた断片的な知識を一つのプログラムとしてまとめるための練習プロジェクトです。
---
##実装内容 
·Swing を使用した基本的なウィンドウ表示および描画処理
·Timer を使用したゲーム状態の定期更新
·LinkedList<Node> を用いたヘビの身体座標の管理
·エサを食べた際に身体が長くなる処理の実装
·壁または自身に衝突した場合のゲーム終了判定
·KeyAdapter によるキーボード入力の取得と移動方向の制御
---
##工夫した点
·ヘビの身体とエサを同じ Node クラスで表現することで、座標の扱いを統一し、処理を分かりやすくしました。
·ヘビの移動処理では、頭部にノードを追加し、末尾のノードを削除することで前進を表現しています。
·また、エサを食べた場合には、頭部にノードを追加することで身体が長くなるようにし、処理の流れが直感的に理解できるようにしました。
---
##苦労した点
実装の初期段階では、どのようなクラスを作成し、各クラスにどのようなメソッドを用意すればよいかは理解していましたが、実際に作成を進める中で、メソッドを記述する順序が混乱し、処理の流れが分かりにくくなることがありました。
その後、処理内容を整理しながら何度も修正を行うことで、各クラスの役割を徐々に分離し、ゲーム全体のコード構造をより分かりやすくすることができました。
---
##今後の改善点
·今後は引き続き実装経験を積み、「まず実現したい機能を整理し、その後に処理の流れを考え、最後にコードを書く」という思考の流れを身に付けていきたいと考えています。
·また、コード構造が分かりやすいかどうかを意識しながら実装することで、可読性の高いプログラムを書く習慣を身に付けたいです。
